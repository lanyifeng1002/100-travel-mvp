# 产品需求

## 1. 项目名称

Amazing 100 Journeys

## 2. 项目背景

传统旅游内容平台普遍存在三个问题：内容同质化严重、推荐逻辑偏热门景点、用户很难快速判断某个目的地是否真正适合自己。尤其在“发现灵感”阶段，用户面对的是碎片化信息，而不是可比较、可筛选、可继续深入的目的地内容。

本项目希望完成一个轻量级目的地展示平台，用较低的交互成本帮助用户完成以下动作：

- 浏览一批风格化目的地
- 按地区、风格、关键词缩小范围
- 进入详情页判断是否值得加入旅行清单
- 在管理后台持续维护和新增内容

## 3. 产品定位

这是一个偏“目的地内容展示”的 Web 应用，不是机票酒店预订平台，也不是完整旅游社区。

核心定位：

- 面向旅行灵感发现阶段
- 强调内容浏览和判断效率
- 适合做轻量级内容型产品

## 4. 目标用户

主要面向以下三类用户：

1. 想寻找新旅行方向的年轻用户

- 经常浏览旅行内容
- 对千篇一律的热门推荐有疲劳感
- 希望看到更有个性、更易比较的目的地信息

2. 注重视觉体验和内容质感的用户

- 喜欢高质量图片和有节奏感的页面
- 更愿意先被内容吸引，再决定是否深入规划

3. 内容维护者

- 需要持续补充新的目的地条目
- 希望录入后能快速在前台生效
- 需要统一字段结构，避免内容展示不一致

## 5. 用户痛点

- 很多平台内容以“热门景点”罗列为主，缺少可筛选、可对比的入口
- 用户往往不知道“这个地方适不适合我”，而不是“不知道这个地方叫什么”
- 内容更新成本高，新增条目后不能快速形成可展示页面
- 前台和后台脱节，内容维护和页面展示不在同一闭环里

## 6. 核心需求

### 6.1 前台展示

- 展示一批精选目的地
- 每个目的地包含名称、地区、图片、最佳时间、时长、预算和摘要
- 支持点击进入详情页

### 6.2 发现与筛选

- 支持关键词搜索
- 支持标签筛选
- 支持随机探索，帮助用户发现未明确目标时的新方向

### 6.3 详情判断

- 展示完整介绍
- 展示亮点、适合人群、出行提醒
- 给出相关推荐，帮助用户继续比较

### 6.4 内容管理

- 提供录入页面
- 支持新增目的地
- 保存后前台可以直接读取到新数据

## 7. 非目标

以下内容不在当前版本范围内：

- 用户注册登录
- 收藏、评论、点赞
- 订单、支付、预订
- 内容审核流
- 多角色权限系统

## 8. 技术要求

- 前端使用现代 Web 框架实现
- 后端提供真实 API
- 数据使用 SQLite 持久化
- 项目可本地运行、可构建、可测试

## 9. 当前版本结果

当前版本已经完成：

- Vue 前台页面
- Express API
- SQLite 数据库存储
- 100 条种子数据
- 内容管理页
- 自动化测试
- Docker 部署文件

## 10. 后续迭代方向

- 增加后台编辑和删除能力
- 增加图片上传
- 增加用户系统与权限控制
- 增加线上部署与独立数据库
