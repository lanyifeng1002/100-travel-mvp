# 项目修改指南

## 1. 先理解项目结构

下面这些目录和文件，可初步理解：

- `src/views/Home.vue`
- `src/views/Detail.vue`
- `src/views/Admin.vue`
- `src/components/NavBar.vue`
- `src/api/travelApi.js`
- `server/index.js`
- `server/repositories/travelRepository.js`
- `server/db/initDatabase.js`
- `src/data/travels.js`
- `tests`

## 2. 后续各部分分别可以改什么

### 2.1 改首页展示

文件：

- `src/views/Home.vue`

适合改的内容：

- 首页标题和文案
- 搜索与筛选交互
- 主推卡片展示方式
- 空状态和加载状态

### 2.2 改详情页展示

文件：

- `src/views/Detail.vue`

适合改的内容：

- 详情文案结构
- 亮点、适合人群、出行提醒布局
- 相关推荐展示样式

### 2.3 改品牌和整体视觉

文件：

- `src/components/NavBar.vue`
- `src/assets/main.css`

适合改的内容：

- 项目名称
- 头部说明文案
- 主色、字体、圆角、阴影
- 全局间距与背景

### 2.4 改目的地数据

文件：

- `src/data/travels.js`

适合改的内容：

- 种子数据内容
- 目的地标题、标签、亮点
- 生成逻辑

注意：

- 如果改了字段结构，同时要同步改数据库初始化和后端映射

### 2.5 改数据库结构

文件：

- `server/db/initDatabase.js`

适合改的内容：

- 数据表字段
- 初始化逻辑
- 重置逻辑

如果新增字段，还要一起改：

- `server/repositories/travelRepository.js`
- `src/api/travelApi.js`
- 对应页面组件

### 2.6 改接口逻辑

文件：

- `server/index.js`
- `server/repositories/travelRepository.js`

适合改的内容：

- 新增接口
- 查询条件
- 表单校验规则
- 推荐逻辑

### 2.7 改管理后台

文件：

- `src/views/Admin.vue`

适合改的内容：

- 表单字段
- 成功和失败提示
- 最近更新列表

## 3. 最快熟悉项目的方法

建议按这个顺序：

1. 执行 `npm install`
2. 执行 `npm run db:reset`
3. 执行 `npm run dev`
4. 在浏览器打开首页、详情页和 `/admin`
5. 对照页面去读 `Home.vue`、`Detail.vue`、`Admin.vue`
6. 再读 `src/api/travelApi.js`
7. 再读 `server/index.js`
8. 最后读 `server/repositories/travelRepository.js`

这样你会先看到“页面效果”，再看到“请求怎么发”，最后看到“数据怎么查”，更容易形成完整理解。

## 4. 最适合练手的后续修改任务

如果想快速熟悉，不要一上来大改架构。先做下面这些小改动：

### 练手任务 1

在 `src/data/travels.js` 里改一个目的地标题或标签，然后执行：

```bash
npm run db:reset
npm run dev
```

观察首页和详情页是否同步变化。

### 练手任务 2

在 `src/views/Home.vue` 里改首页主标题、按钮文案和统计卡片文案，熟悉页面结构。

### 练手任务 3

在 `src/views/Admin.vue` 里新增一个表单提示语，理解后台录入和前台数据之间的关系。

### 练手任务 4

运行：

```bash
npm test
```

再去看 `tests` 目录，理解哪些逻辑已经被验证过。

## 5. 每次修改后的正确操作

如果改了前端页面：

```bash
npm run dev
```

如果改了种子数据：

```bash
npm run db:reset
npm run dev
```

如果改了接口或数据库字段：

```bash
npm run db:reset
npm test
npm run build
```

## 6. 提交前自查

每次准备交付前，至少跑这三条命令：

```bash
npm run db:reset
npm test
npm run build
```

只要这三步都通过，项目整体结构通常就是稳定的。
