## 三、老师要求对应关系

1. 完整项目源码

- 已提供前端、后端、数据库初始化脚本、测试、部署文件

3. 开发过程思路与工作流说明

4. 需求文档与 API 文档

- 已提供 `docs/01-需求文档.md`

5. README

- 已提供根目录 `README.md`
- 包含运行指南、技术选型说明、页面结构和后续扩展方向

## 四、提交前建议

- 使用源码目录打包，不要把 `node_modules` 一起提交
- 如果不是提交构建产物，也不要把 `dist` 作为重点交付内容
- 如果需要邮箱附件，建议压缩包命名为 `姓名-AtlasJourneys-源码.zip`
- Prompt 文档和开发说明文档可以单独导出 PDF，也可以直接保留 Markdown
